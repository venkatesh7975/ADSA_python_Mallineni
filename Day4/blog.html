Strings, Anagrams & Recursion

Deep dive into algorithm problem solving and mastering the Call Stack.

Today's Topics

  • Built-in String Methods (`strip`, `split`, `replace`)
  • Solving Anagrams (Sorting vs Frequency Dictionary)
  • Advanced Recursion (Winding vs Unwinding)
  • Call Stack Visualization & Dry Runs
  • Time & Space Complexity Basics

Key Concepts

1. String Methods

Python provides powerful built-in methods to manipulate text.

text = "  Hello World!  "
clean = text.strip().lower()       # "hello world!"
words = clean.split()              # ["hello", "world!"]

2. The Recursion Call Stack

Understanding recursion means understanding the stack frames that build up in memory.

def factorial(n):
    if n <= 1: return 1            # Base Case
    return n * factorial(n - 1)    # Recursive Step

Detailed Quiz (20 MCQs)

Test your knowledge on Strings, Anagrams & Recursion!

1. What does the `.strip()` method remove?

  • A) All spaces in the string
  • B) Leading and trailing whitespace
  • C) Punctuation
  • D) Vowels
View Answer

Answer: B (It cleans up the ends of a string, useful for user input)

2. What does `.split(",")` return?

  • A) A tuple of characters
  • B) A list of substrings separated by commas
  • C) A new string without commas
  • D) A dictionary
View Answer

Answer: B (`split()` breaks a string into a list based on the provided delimiter)

3. How does the `.join()` method work?

  • A) `list.join("-")`
  • B) `join(list, "-")`
  • C) `"-".join(list)`
  • D) `list + "-"`
View Answer

Answer: C (`join` is called on the separator string and passed the iterable)

4. What is the difference between `.find()` and `.index()`?

  • A) `.find()` is for lists, `.index()` is for strings
  • B) `.find()` returns -1 if not found; `.index()` raises a ValueError
  • C) `.find()` searches backwards
  • D) They are identical
View Answer

Answer: B (Use `.find()` if you aren't sure the substring exists to avoid crashes)

5. Does `"aaaa".count("aa")` return 2 or 3?

  • A) 3
  • B) 2
  • C) 4
  • D) 0
View Answer

Answer: B (The `.count()` method only counts non-overlapping occurrences)

6. What defines an anagram?

  • A) Strings that read the same backwards (e.g., racecar)
  • B) Strings formed by rearranging the same letters (e.g., listen/silent)
  • C) Strings with the same length
  • D) Strings containing only unique characters
View Answer

Answer: B (They must have the exact same character frequencies)

7. What is the time complexity of checking anagrams by sorting both strings?

  • A) O(1)
  • B) O(N)
  • C) O(N log N)
  • D) O(N^2)
View Answer

Answer: C (The sorting algorithm dominates the time taken)

8. What is the time complexity of checking anagrams using a frequency dictionary?

  • A) O(1)
  • B) O(N)
  • C) O(N log N)
  • D) O(N^2)
View Answer

Answer: B (You only need to iterate through the strings a constant number of times)

9. Which standard library module is excellent for checking character frequencies?

  • A) math.Freq
  • B) collections.Counter
  • C) itertools.count
  • D) string.frequency
View Answer

Answer: B (`Counter(s1) == Counter(s2)` is a fast one-liner for anagram checks)

10. What is the Call Stack?

  • A) A list of variable names
  • B) A memory structure that tracks active function calls
  • C) A database of errors
  • D) A queue for printing text
View Answer

Answer: B (It pushes a frame when a function is called and pops it when it returns)

11. What causes a Stack Overflow (RecursionError)?

  • A) Too many variables
  • B) Infinite loop using `while`
  • C) Missing or incorrect base case in a recursive function
  • D) Using large numbers
View Answer

Answer: C (The call stack exceeds its maximum depth limit)

12. What is a Base Case?

  • A) The starting point of the function
  • B) The condition that stops the recursion
  • C) The deepest part of the call stack
  • D) The default argument value
View Answer

Answer: B (It prevents infinite recursion)

13. What is the Recursive Step?

  • A) When the function returns a value
  • B) When the function prints something
  • C) When the function calls itself with a modified argument
  • D) When the function hits the base case
View Answer

Answer: C (e.g., `return n * factorial(n - 1)`)

14. What is the "winding phase" of recursion?

  • A) When values are being returned up the stack
  • B) When frames are being pushed onto the stack before hitting the base case
  • C) When the loop is spinning
  • D) When memory is cleared
View Answer

Answer: B (The top-down phase of calling)

15. What is the "unwinding phase" of recursion?

  • A) When values are returned and frames are popped off the stack
  • B) When the base case is hit
  • C) When the program crashes
  • D) When arguments are modified
View Answer

Answer: A (The bottom-up phase of returning values)

16. What shape is the recursion tree for standard Factorial `fact(n)`?

  • A) A binary tree (branching in two)
  • B) A linear line straight down
  • C) A circular graph
  • D) A grid
View Answer

Answer: B (It only makes one recursive call per frame, so it does not branch)

17. What shape is the recursion tree for naive Fibonacci `fib(n)`?

  • A) A binary tree branching rapidly
  • B) A linear line straight down
  • C) A circular graph
  • D) A grid
View Answer

Answer: A (It makes two calls per frame, e.g., `fib(n-1) + fib(n-2)`)

18. What is an "overlapping subproblem" in a recursion tree?

  • A) When two functions have the same name
  • B) When the exact same function call (e.g., `fib(2)`) is calculated multiple times independently
  • C) When the stack limit is reached
  • D) When variables are shadowed
View Answer

Answer: B (This is what makes naive Fibonacci incredibly slow: O(2^N))

19. What is Memoization?

  • A) Storing variables in a list
  • B) Caching the results of expensive function calls to avoid recalculating overlapping subproblems
  • C) Using memory efficiently
  • D) Converting recursion to iteration
View Answer

Answer: B (Often implemented with a dictionary or `@functools.lru_cache`)

20. What is Tail Recursion?

  • A) When the base case is at the bottom
  • B) When the recursive call is the very last operation performed, with no subsequent calculations pending
  • C) When recursion is used to loop backwards
  • D) When recursion returns a list
View Answer

Answer: B (Note: Python does not optimize tail recursion, unlike C++ or Scheme)
