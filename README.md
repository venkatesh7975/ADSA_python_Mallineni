# Python for ADSA Course 🚀

Welcome to the Python fundamentals course repository! This folder serves as the central hub for all class materials, notes, scripts, and interview preparation guides. 

As we progress through the course, new daily folders (like `Day1`, `Day2`, etc.) will be added here containing the specific materials for those sessions.

---

## 📅 Course Outline & Daily Summaries

### [Day 1: Python Fundamentals, Variables, Data Types & Operators](./Day1/)
**Goal of the day:** Introduce Python from scratch, build confidence with basic syntax, understand how data is stored in memory, and connect these concepts to simple data analysis tasks.

**What you will learn in the Day 1 folder:**
*   **What is Python?** Why it's heavily used in data analysis, automation, and backend development.
*   **Basic Syntax:** How to write comments, use indentation correctly, and print to the console.
*   **Variables:** How to declare variables, naming conventions (like `snake_case`), and understanding dynamic typing.
*   **Data Types:** Working with numbers (`int`, `float`), text (`str`), booleans (`bool`), and collections (`list`, `tuple`, `dict`, `set`).
*   **Operators:** Performing math (arithmetic), comparing values (comparison), and combining conditions (logical operators).

**Key Highlights Inside the `Day1` Folder:**
1.  **`README.md`**: Start here! It contains links to all Day 1 materials and hands-on tasks for you to practice.
2.  **Detailed Notes**: Broken down by topic (`variables.txt`, `datatypes.txt`, `operators.txt`) with real-world analogies.
3.  **Executable Scripts**: Python files (`.py`) that you can run on your machine to see the code in action.
4.  **Mini-Project**: A `sales_analysis.py` script that brings all Day 1 concepts together in a practical scenario.
5.  **100 Interview Questions**: A comprehensive guide to test your knowledge and prepare for job interviews.

### How to Use This Repository
1.  Navigate to the specific day's folder.
2.  Read the detailed notes (`.txt` files) to understand the theory.
3.  Open the Python scripts (`.py` files) in your preferred IDE (like VS Code or PyCharm) and run them.
4.  Try changing the values in the scripts to see how the output changes.
5.  Complete the tasks assigned in each day's local `README.md` file!
